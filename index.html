// 01   overview

What this project does

This lab provisions three Ubuntu 22.04 virtual machines using Vagrant and VirtualBox, then uses Ansible to configure the two managed nodes automatically — all from a single command. It demonstrates core DevOps skills: infrastructure as code, configuration management, SSH key distribution, and idempotent provisioning.

⚙️

Automated Provisioning

SSH keys are generated and distributed across VMs entirely inside the provisioner — no host-side setup required.

🔁

Idempotent Playbooks

Ansible roles can be re-run safely at any time. Only changed state triggers an update.

🔒

Key-Based SSH Only

Password authentication is disabled on managed nodes. Access is via a generated RSA key on the control node.

🖥️

Cross-Platform

Works on Windows, macOS, and Linux. No host-side ssh-keygen or file reads at Vagrantfile parse time.

// 03   setup

Getting Started

Requirements: Vagrant ≥ 2.3 and VirtualBox ≥ 7.0 installed on your machine.

terminalbash
# 1. Install the required Vagrant plugin (once)
$ vagrant plugin install vagrant-hostmanager

# 2. Boot all three VMs and run provisioning
$ vagrant up

# 3. SSH into the control node
$ vagrant ssh ansible

# 4. Ping all machines
vagrant@ansible:~$ ansible all -m ping
ansible | SUCCESS => { "ping": "pong" }
pc1     | SUCCESS => { "ping": "pong" }
pc2     | SUCCESS => { "ping": "pong" }

Useful Commands

ansible — ad-hoc & playbookbash
# Run the full playbook
$ ansible-playbook ~/ansible/site.yml

# Dry run (no changes applied)
$ ansible-playbook ~/ansible/site.yml --check

# Ad-hoc commands
$ ansible managed_nodes -m shell -a "uptime"
$ ansible managed_nodes -m shell -a "df -h"
$ ansible pc1 -m shell -a "hostname -I"
